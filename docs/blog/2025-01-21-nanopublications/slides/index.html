Cyberspace

  • ¬© William Gibson
  • A set of all quads available
    for an individual
  • Quad is:
    • Subject
    • Predicate
    • Object
    • IRI of the source where the above where obtained
For the purposes of this presentation, I propose the term "cyberspace" to denote the set of all RDF quads available to an individual, the whole volume of machine readable data a particular person or machine can access. We will also assume that the named graph in each of those quads signifies the source from which the quad is retrievable. We are a hijacking a term coined by William Gibson, a sci-fi writer, and I also hope he will forgive us.

I've written a little tool, named Iolanta, that might assist in navigating the vast depths of the cyberspace. I apologize if the LLM-generated backgrounds on my slides were hinting on something colorful and three-dimensional, and vaguely futuristic, - Iolanta is nothing of the sort, it is merely a terminal application.

Now this is a bit more interesting. This nanopublication asserts a few triples about red things, which are a class, have a label and a definition. All written by Tobias again, and then retracted using the publication we have already seen.

Let's click on the label. From the header of the screen, one might guess that this is a regular label property from RDF Schema vocabulary, but this page is trying to make it more explicit. We can see that RDFS defines this term and we can jump to the whole vocabulary.

Which conveniently shows us its terms, grouped by rather arbitrary categories. This is only one of possible views for the RDFS ontology, it is designed to present it, well, as ontology - that is why Iolanta chooses it as a default. However, there are other views, this is the Properties view which describes the ontology itself.

Note that one of its properties is Term Group, and that's exactly what defines grouping of terms that we have been looking at. This term is defined by VANN ontology by Ian Davis back in 2010.

Use Cases

  • Browse the Cyberspace
  • Generate tables, lists, diagrams, and more from Cyberspace
  • Inject Cyberspace-sourced information into human readable content
  • Ideas are welcome!

What is Iolanta good for? Why is it built? - In order to make machine readable data more readable for a human. Now opinions might differ as to how human friendly a terminal is, but as a programmer, I use terminal a lot, and this environment was a natural habitat for me to experiment within.

In addition, terminal gives you quite a lot of interoperability. At this point, Iolanta is drawing a text based user interface, but it will be even easier to generate JSON, YAML, HTML, TeX or any other textual format - based on Linked Data available in the Cyberspace.

Therefore, in addition to merely browsing the Cyberspace, retrieval of information from there for use in other programs, or as part of documents, is also an important use case.

I will be thrilled to hear your ideas of how it might be useful in your work.
